SDuX Vault Debugger Example
This example demonstrates the SDuX Vault built-in debugger — a floating panel that captures pipeline execution traces. Record a session, trigger state changes, then export logs or generate an AI diagnostic report.
1. Enable Dev Mode in Vault Definition
The debugger is enabled via devMode: true in the Vault configuration. This example already has it enabled.
File: app/app.config.ts
2. Add insights to the FeatureCell Definition
Insights provide additional context for the debugger to capture during pipeline execution. This example already has insights enabled.
File: app/app.config.ts
3. Record a Session
To record a FeatureCell session, click the "Record" button, then trigger state changes by clicking "Load Sample State" in the UI. The debugger captures each pipeline event, including filter and reducer executions, state emissions, and any errors.
After finishing all state changes, click "Stop" to end the recording session. You can then download the captured logs as a JSON file for analysis or use the AI Assist feature to generate an automated diagnostic report.
Control Description
Record / Stop Start or end a recording session
Event Count Total captured pipeline events
Error Count Captured error signals
Clear Reset the current session
Download Logs Export the debug dump (JSON)
AI Assist Generate an AI diagnostic report
Create Issue Generate a structured issue report
Note: This example uses a filter (removes "Han") and a reducer (marks Jedi). See app/example.service.ts for the pipeline configuration.
